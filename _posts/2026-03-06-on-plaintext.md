title: on plaintext
---

there is something to be said for plain text. no fonts to choose, no colours to agonise over, no layout to break on some obscure screen size. just characters on a page.

most of the web is buried under layers of abstraction. frameworks on top of frameworks, each one promising to make things simpler, each one adding its own weight. a page that could be a few kilobytes arrives as several megabytes. the content is still the same few paragraphs it always was.

plaintext forces a kind of honesty. you cannot hide a weak argument behind a beautiful gradient. the words have to do the work.

this is not nostalgia for a worse time. it is a preference for tools that stay out of the way. a text file opens instantly, on anything, forever. it does not require a runtime or a build step. it does not phone home. it does not ask you to accept cookies.

the fixed-width column is also not an accident. somewhere around 66 to 72 characters per line is where reading gets comfortable. the eye can travel from the end of one line to the start of the next without losing its place. newspapers figured this out. books figured this out. the web largely forgot.

if you are writing something that is meant to be read, it probably looks better as plaintext than you think.
